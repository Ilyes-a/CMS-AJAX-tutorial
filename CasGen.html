Vue d'ensemble

Le projet permet de connecter un SI indépendant à divers CMS via des appels dynamiques (fetch()). Deux scénarios se présentent :

  • Environnements multi-VM : Chaque CMS et le SI sont installés sur des machines virtuelles distinctes avec des adresses IP spécifiques. Le router du SI adapte les chemins en fonction des IP.
  • Serveur unique : Tous les systèmes partagent la même IP. L'utilisation de sous-domaines (ex. wordpress.example.com, joomla.example.com) permet de différencier les CMS et d'ajuster dynamiquement les routes.

Adaptations Techniques

Pour les environnements multi-VM, le router détecte le CMS selon la terminaison de l'IP (ex. "35" pour WebSiteBaker, "22" pour WordPress). Pour un serveur unique, des sous-domaines distincts ou une configuration explicite (variables d'environnement ou fichier de config) permettent de définir correctement ROUTE_PREFIX et ROUTE_SUFFIX.

Cette approche garantit une intégration interopérable, sécurisée et performante entre le SI et les CMS.
